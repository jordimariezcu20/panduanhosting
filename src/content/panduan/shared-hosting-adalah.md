**Shared hosting** adalah jenis layanan hosting di mana satu server fisik digunakan bersama oleh banyak website sekaligus — bisa ratusan bahkan ribuan website dalam satu server yang sama. Setiap website berbagi resource server yang sama: CPU, RAM, bandwidth, dan storage.

Ini adalah jenis hosting paling populer dan paling terjangkau, menjadi pilihan utama bagi pemula yang baru memulai website.

## Cara Kerja Shared Hosting

Bayangkan sebuah apartemen besar. Satu gedung (server) dibagi menjadi banyak unit (website). Semua penghuni berbagi fasilitas umum yang sama: listrik (daya server), air (bandwidth), dan area parkir (storage).

Setiap website mendapatkan:
- Direktori penyimpanan sendiri yang terisolasi
- Akses ke database (MySQL)
- Panel kontrol (biasanya cPanel) untuk mengelola file, email, dan pengaturan
- Quota storage dan bandwidth sesuai paket

Yang dibagi bersama adalah resource komputasi server (CPU dan RAM). Jika salah satu website mengalami lonjakan traffic ekstrem, website lain di server yang sama bisa terpengaruh.

## Kelebihan Shared Hosting

### 1. Harga Paling Terjangkau
Ini alasan utama shared hosting begitu populer. Biaya server dibagi banyak pengguna, sehingga setiap pengguna hanya membayar sebagian kecil.

## Kekurangan Shared Hosting

### 1. Resource Terbatas dan Bisa Tidak Stabil
Karena berbagi server, resource yang tersedia untuk website kamu dibatasi. Jika website tetangga di server yang sama mengalami lonjakan traffic atau memakai resource berlebihan, performa website kamu bisa terdampak — fenomena ini disebut "noisy neighbor effect".

Provider hosting berkualitas menerapkan pembatasan per akun agar ini tidak terjadi, tapi tetap ada batasnya.

### 2. Tidak Cocok untuk Traffic Tinggi
Shared hosting memiliki batas kemampuan menangani pengunjung bersamaan. Jika website kamu mulai mendapat ratusan ribu pengunjung per bulan, shared hosting mungkin tidak lagi memadai.

### 3. Kontrol Terbatas
Kamu tidak bisa menginstall software server sendiri, mengubah konfigurasi PHP server-wide, atau melakukan hal-hal yang membutuhkan akses root. Semua dibatasi oleh konfigurasi yang ditetapkan provider.

### 4. Risiko Keamanan Bersama
Jika ada website lain di server yang sama terkena malware atau serangan, ada kemungkinan (kecil) dampak ke website tetangga. Provider hosting yang baik mengisolasi setiap akun dengan baik untuk meminimalkan risiko ini.

## Kapan Harus Upgrade dari Shared Hosting?

Tanda-tanda kamu perlu naik level:

- **Website sering lambat** meski sudah dioptimasi
- **Traffic melebihi 50.000 pengunjung/bulan** secara konsisten
- **Error 500 atau timeout** yang sering terjadi
- **Butuh akses root** atau software khusus yang tidak tersedia di shared hosting
- **Aplikasi membutuhkan resource tinggi** (video streaming, machine learning, dll.)

Jika salah satu tanda di atas muncul, pertimbangkan upgrade ke **VPS hosting** atau **cloud hosting**.

## Tips Memilih Paket Shared Hosting

**1. Jangan tergiur unlimited**
"Unlimited storage" dan "unlimited bandwidth" selalu ada batasnya di fine print. Perhatikan juga batas penggunaan CPU dan jumlah proses yang diizinkan.

**2. Pilih SSD, bukan HDD**
SSD (Solid State Drive) jauh lebih cepat dari HDD tradisional. Semua hosting modern yang bagus sudah menggunakan SSD.

**3. Cek lokasi server**
Untuk website dengan pengunjung mayoritas Indonesia, pilih server di Indonesia atau Singapura. Latency yang rendah = loading lebih cepat.

**4. Perhatikan limit database**
Beberapa paket murah membatasi jumlah database MySQL. Jika kamu berencana install banyak website atau aplikasi, pastikan tidak ada batasan ketat.

**5. Cek kebijakan refund**
Provider yang percaya diri dengan produknya biasanya menawarkan garansi uang kembali 30 hari. Ini tanda bahwa mereka yakin kamu akan puas.

## FAQ: Shared Hosting

**Berapa website yang bisa di-hosting di shared hosting?**

Tergantung paket. Paket dasar biasanya untuk 1 domain (+ subdomain), paket menengah untuk 5–10 domain, dan paket premium sering menawarkan "unlimited website/domain". Pastikan cek dulu batas di paket yang kamu pilih.

**Apakah shared hosting aman?**

Ya, jika memilih provider yang berkualitas. Provider yang baik mengisolasi setiap akun pengguna satu sama lain, memiliki firewall, anti-malware, dan monitoring aktif. Hindari hosting murahan yang mengabaikan keamanan.

**Apakah bisa install aplikasi sendiri di shared hosting?**

Tergantung aplikasinya. WordPress, Joomla, Drupal, dan banyak CMS/aplikasi PHP bisa diinstall. Tapi aplikasi yang membutuhkan akses root, port non-standar, atau software server tertentu tidak bisa diinstall di shared hosting.

**Berapa lama setup shared hosting?**

Sangat cepat — biasanya aktif dalam hitungan menit hingga 1 jam setelah pembayaran. Kamu langsung bisa menginstall WordPress dan mulai membuat konten.

**Apa bedanya shared hosting dengan web hosting?**

"Web hosting" adalah istilah umum untuk semua jenis layanan hosting website. Shared hosting adalah salah satu jenis web hosting (selain VPS, cloud, dedicated). Sering kali ketika orang menyebut "web hosting", yang dimaksud adalah shared hosting.
